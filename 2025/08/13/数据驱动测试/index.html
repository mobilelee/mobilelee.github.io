什么是数据驱动

相同的测试脚本使用不同的测试数据执行，并完全分离了测试数据和测试行为，这样的设计模式叫数据驱动

TestNG 数据驱动

实现数据驱动的两个关键词注解：

1.@DataProvider（主要使用）；

2.@Parameters（是在 TestNG.xml 中写死的注解，并不常使用）；接下来主要讲解@DataProvider的使用

DataProvider 基础

DataProvider 是 TestNG 提供的一个注解，返回的是一个对象的二维数组；返回的数组参数个数要与要求的一致，不然会报错；只要按照它规定的方法去使用，就能够建立最简单的你想要的数据驱动模式；在 dataProvider 里面标注 name，然后在 Test 里面写上对应的 dataProvider 的名字，就能调用，其中

这部分在不停得调用数据，能够使一个表达式使用不同的数据去输出，这就是数据驱动模式

注意点：

1、dataProvider 要求返回值必须是 Object[] 或者 Object[][]；在里面传递的返回值也必须是这种类型的；

2、在 Test 方法里面，要设置传入的参数，这个参数个数必须需 retrun 里面的 Object 数值个数一致；比如(String company,String people)与 {"TesterHome","sihan"},必须个数一致，当单独修改为(String company,String people,String adress)或者{"TesterHome","sihan","sichuan"}都不行，TestNG 会报错，必须两者的个数一致

DataProvider 扩展

扩展：DataProvider 返回的是一个 Object 数组，也就是说不管我以何种方式的文件传输进来，只要我进行格式解析，并且最后将里面的数据以 Object 形式返回满足 DataProvider，就可以实现外部填写数据进行数据驱动
