变量定义

在 bash 中等号左右不要有空格；符合人写的手的手工定义；定义字符串时可以双引号，可以单引号，也可以不加引号，但是内容中包含空格一定要加单引号或者双引号；在 shell 里面命名没有那么严格。不被赋值的变量就显示为空；赋值的时候单引号和双引号有区别，要注意的点:单引号不会将值传递而双引号会传递

变量使用

echo ${b}比 echo $b 更为严谨

预定义变量： 有一些变量为系统使用，并且已经帮我们预先定义好了，我们只需要使用即可

tips: echo $PWD 和 pwd 都会打印路径；区别：前者是一进入的时候就会赋值，只要打印即可；后者是一个命令，需要启动进程去执行，比较耗费性能，推荐用第一种

shell 的特性： 任何语句的执行都会返回一个状态值，输入 echo $?就可以查看状态值是否为；0 为真，非 0 为假

Bash 环境

  • bash 是一个进程

  • bash 下还可以再启动一个 shell，即子进程，这个 shell 是 sub shell，原 shell 会复制自身给他；可以不停得在创建子进程；想要推出就用 exit，依次退出子进程；直至到最外面到 shell 进程

  • 在 sub shell 中定义的变量，会随着子进程到消亡而消失

  • (a=testerhome;echo $a) # 打印结果为 testerhome；但是只有这一次，当执行完之后，a 就会被销毁，不存储

  • {a=testerhome;echo $a} # 打印结果为 testerhome；但是变量还在，当再次去打印 echo "$a"的时候，就会打印出 testerhome

  • echo $$ # 获得当前脚本执行的 pid（进程 id）

  • & # 代表放置后台执行，例如 sleep 30&就是将 sleep30 这个操作放置后台执行

  • echo $! # 打印正在后台运行的最后一个作业的 PID（进程 pid）

  • Shell 首先是一个工作环境，可以有很多变量提供我们使用

  • set 可以获取当前的所有变量，而 evn 也是获得变量，只不过变量是可以传递给子进程的；以下是部分截图

  • export aa=bbbb 把私有变量导出，让子进程也能过访问；要在同一个窗口，export 的效力仅限于该次登陆操作，也就是只对该 shell 和子 shell 有效，shell 结束时变量消失

Shell 输入输出

Read # 用来读取输入，并赋值给变量；自己的理解：一般用于函数中，写在 sh 脚本中执行；从标准的输入中读取一行，并把每行的字段值给指定的 Shell 变量

echo ，printf 可以简单的输出变量；echo 打印换行，printf 不换行

| 表示管道，也就是前一个命令的输出传入下一个命令的输入

脚本编写

借助于写脚本，可以把所有的逻辑保存下来，同时可以传参，实现脚本定制化；通过传不同的参数进去，让脚本执行不同的数据

输出内容分别是 $0 就是当前执行时候的脚本的名字，相对于当前路径而言,$1 是传入的第一个参数，这边是传入的路径,$2 传入的第二个参数，一共能接受 9 个,$@ 和 $都表示全部参数：$@ 在传入的参数包含空格的时候，能将空格保留，就是完整的一整个参数,$会将参数中的空格去除，分解，例如下面都是打印 b c；但是 $@ 表示的是 “b c”;但是 $已经将内容拆散了，变成 b 和 c，所所以平时用参数的时候尽量使用 $@

在 $@ 和 $里面加入 for 循环，来查看结果，发现 $@ 更加符合我们所理解的参数含义，因此推荐使用 $@

函数 可以组织所有的逻辑代码语言，当 shell 脚本里面的逻辑代码变多的时候，写在函数内去执行，是一样的效果；写的时候先定义一个函数名称 find_all_file();然后回车，输入{逻辑代码块}；最后回车表示写完，然后直接输入 find_all_file() 就会输出执行结果

同样函数也能传参,可以借助函数去改变脚本的设计，让脚本阅读起来更加清晰 ,输出的结果内容是一致的

一个脚本承载若干个函数，每个函数包含小的功能点，就能完成一次大的脚本包装编写

脚本执行的变量保留 ：bash 去执行脚本的时候，是在子进程里面执行的，所有完成之后会将变量销毁，而 source 去执行的时候就会将变量都保留到当前进程，所以最后去打印的时候会在当前进程中打印出来，source 还有一种显示方式是".";.3.sh 效果一样
