UI自动化测试-appnium

发表于2025-08-19|更新于2025-08-19|自动化测试
|浏览量:

工作原理

启动 appium 的 desktop 版本，会自动开启一个端口 4723；可以通过 java-client；Appium-Python-Client 等等的客户端，编写 java 或者 python，ruby 等脚本；http 协议通过 4723 端口传给 appium，appium 用 http 请求通过 4724 端口发送给手机设备来操作手机进行 UI 自动化；各种版本的 client 可以通过 appium 的 github 上找到（地址：https://github.com/appium）用 Maven 配置下载依赖；其中安卓手机要用过 adb 命令安装启动对应 app；iOS 后面继续补上；inspector 工具是 一个 app 的分析界面，可以讲该 app 界面的所有元素都标出来供使用，本质上也是通过模拟脚本去发请求给 4723 端口再通过 appium 进行操作；

Appium 元素定位


这边有一系列的属性，如何去确定唯一定位

1.resource-id

2.content-desc； 一般如果有也可以用来确定唯一定位

3.text 这个元素有可能产生重复，要结合其他是元素；text + index 或者 text + instance 这种组合来确定

4.instance：实例，表示在全局中同类型实例中排第几，一般用 class + instance 来确定

5.xpath，在 appium 中寻找的时候首先 appium 遍历生成一个页面 xml，再用 xpath 去寻找，所以很耗时，但是定位可以做到准确

录制脚本


点击录制按钮开始录制脚本，选择左侧的控件，然后可以操作右上角的三个选项分别进行点击，输入文本内容，例如登陆注册；清除：清除文本框所有的默认输入等；停止录制就会生成一段代码，有很多语言可选，Java，Python。。。。。

将复制下来的代码 paste 去 idea 等工具，运行 Run 即可通过脚本来模拟 app 等 UI 自动化

进阶API

1.launchApp() # 启动 app，也可以理解为打开 app

2.runAppBackground() # 在括号内可以设定时间，表示把 app 放置后台执行直到 xx 秒之后会重新打开 app

3.sendSMS() # 发短信

4.MakeGSMCall() # 这个可以模拟打电话，在里面设置参数可以选择不接，或者接起来；这个跟上面那个发短信的可以去测试 app 在各种情况下的运行情况；譬如在使用 app 的时候突然来了电话 app 是怎么运行的；

5.closeApp() # 关闭 app

6.lockApp() # 在 app 运行的时候锁定

7.unlockApp() # 解锁手机，再次展现 app

8.toggleAirplaneMode() # 开启飞行模式

9.rotate # 用于横竖屏切换

10.setNetworkSpeed() # 限制网络状态，可以设置弱网情况

11.TouchAction # 很有用的 API，可以用来做手势密码，滑动操作等等，需要细看（等补充）
https://github.com/appium/appium/blob/master/docs/en/about-appium/api.md

关于 appium 的更多 API 可以在 github 里面读，英文文档去读一遍都会很了解，缺什么去那里找就可以了
