原理

selenium 提供了 RemoteWebDriver，可以使执行测试的机器和发送测试命令的机器独立开来，比如 A 机器，是写测试代码的机器，可以发送测试命令给服务器，可以理解为客户端；B 机器是执行测试代码的机器，可以理解为服务器端，A 发送命令给 B，然后再 B 上面执行测试，并返回结果给 A。此外，RemoteWebDriver 还可以调用不同的 driver，譬如 chromedriver、iedriver。firefoxdriver 等等，这样子不用捆绑，更加适合单独执行
需要在执行代码的机器上先下载好 selenium 的 jar 包，在终端执行

ChromeDriver 机制

Java 或者 Python 写好的脚本会传一个 webdriver session 给 selenium、appium，然后 selelnium 或者 appium 再同样穿一个 wendriversession 给 chromedriver，chromedriver 通过 Devtools 协议进行 chrome 浏览器的交互；电脑之间的进程可以随意访问，浏览器会对外展现一个接口，供各种 driver 驱使，譬如 chromedriver；手机上需要额外多通过 usb 数据线将手机上的进程映射到外面来，供 chromedrive 进行访问

DevTools

一个工具，可以再浏览器内打开

  • devices 可以查看当前的设备连接，以及里面的 browser 或者 webview 的页面
  • pages 是展现当前浏览器打开的网页，点击 inspector 都可以进入调试界面
